## 报名材料（问卷 + 申报书）

- [ ] 参赛者基本信息（姓名、联系方式、常用邮箱——及时查收审核邮件）

- [ ] 一页左右 PDF 项目申报书，内容包括：
  - [ ] 项目现有基础说明

  - [ ] （移植项目）原项目、原项目链接、许可证、本次移植范围

## 项目类型（四选一，本项目为「移植」）

1. 原创 MoonBit 开源库
2. **将其他语言生态的成熟开源库移植到 MoonBit** ← 本项目
3. 围绕实际开发需求的开发工具
4. MoonBit 示例工程

约束：

- 避免与 MoonBit 生态中已有的成熟项目功能高度重合
- 以 MoonBit 为主要实现语言（允许必要的绑定代码、配置、构建脚本）
- 参考规模 4,000~10,000 行有效 MoonBit 代码（非硬性标准）

## 移植项目特别要求（必须注明）

- 原项目名称：d3-force
- 原项目链接：https://github.com/d3/d3-force
- 原项目许可证：ISC（OSI 认可，宽松许可证，允许移植）
- 本次移植的具体范围：见 [project.md](project.md) 移植范围章节
- 对原项目所做的修改和适配
- 尚未移植或暂不支持的功能

## 开源与版权红线

- 必须采用 OSI 认可的开源许可证（本项目拟用 Apache-2.0 或 MIT，与 MoonBit 生态主流一致；保留 d3-force 的 ISC 版权声明）
- 不得提交未经授权的私有/闭源/商业代码、来源不明代码
- 不得把他人现有成果直接当原创提交（我们是合规移植，需明确标注来源）
